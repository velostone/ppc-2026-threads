## 1. Контекст

Решается задача выделения ребер на изображении с использованием оператора Собеля. Математическая постановка и
последовательный алгоритм описаны в файле отчета `seq/report.md`.
Алгоритм хорошо распараллеливается ввиду отсутствия зависимостей по данным между итерациями циклов. Предлагается организация
параллелизма с помощью библиотеки TBB.

## 2. Постановка задачи

На вход алгоритма подается изображение типа `Image`, пиксели которого упакованы в `std::vector<uint8_t>`. На выходе
ожидается результирующий вектор `std::vector<int>`, компоненты которого принимают значение 0, если ребро отсутствует,
и `magnitude`, если ребро есть. За деталями обращаться к файлу `seq/report.md`.

## 3. Базовый алгоритм

Последовательная версия алгоритма в цикле по изображению обходит внутренние пиксели, применяя к ним ядра оператора Собеля
и вычисляя нормы соответствующих градиентов. Решение о наличии ребра принимается в результате сравнения значения нормы с
некоторым заданным порогом чувствительности `threshold`. За деталями обращаться к файлу `seq/report.md`.

## 4. Схема распараллеливания

Внешний вычислительный цикл алгоритма был распараллелен с помощью функции `tbb::parallel_for()`. В качестве итерационного
пространства (ИП) передавалось стандартное одномерное ИП `tbb::blocked_range<std::size_t>` с диапазоном итераций
`[1, height - 1)`. `grainsize` при этом явно не задавался, использовалось значение по умолчанию, равное 1.
Вычислительный цикл был обернут в функтор, принимающий в качестве аргумента константную ссылку на ИП, выполняющий
переданный диапазон итераций внешнего цикла, записывающий результат в `output`.
Использовалась стратегия планирования `tbb::simple_partitioner`, пространство итераций расщеплялось рекурсивно до тех пор,
пока не был получен размер порции в 1 итерацию.

Проведенные тесты производительности показали сублинейное ускорение, сохраняющееся для числа потоков, не превышающих
число логических ядер процессора (до 6). При этом в 2 и 4 потока ускорение приблизилось к теоретической верхней границе.
Дальнейшее увеличение числа потоков (более 6) не привело к ожидаемому ускорению. Эффективность всех реализаций с числом
потоков до 6 близка к 1, что говорит о высокой оптимизации создаваемой очереди задач и оптимальном механизме распределения
работы. На 12 потоках эффективность падает до 0.41, поэтому оптимальным соотношением числа потоков к числу логических
ядер остается 1 к 1.
По сравнению с OpenMP параллелизм показал несколько лучшие результаты для 2 и 4 потоков (1.94 против 1.99 для 2 потоков
и 3.77 против 3.95 - для 4), что говорит о преимуществе динамического распределения задач с механизмом кражи перед
статическим распределением в рассматриваемой задаче. Накладные расходы становятся заметны при 6 и более потоках:
TBB реализация показала себя хуже аналогичной реализации OpenMP.

## 9. Выводы

При решении задачи выделения ребер на изображении с помощью оператора Собеля параллелизм, организованный средствами TBB,
дал сублинейное ускорение до 5.48 раз при 6 потоках, что соответствует числу логических ядер процессора.
Наибольшая эффективность 0.99 была достигнута на 2 потоках. Дальнейшее масштабирование привело к падению эффективности
до 0.41 при 12 потоках.
Библиотека сработала лучше OpenMP для небольшого числа потоков (до 4), но показала худший результат для 6 и 12 потоков.

## Приложение

```cpp
// File: tbb/src/ops_tbb.cpp
tbb::parallel_for(tbb::blocked_range<std::size_t>(1, h_ - 1), [&](const tbb::blocked_range<std::size_t> &r) {
  for (std::size_t j = r.begin(); j < r.end(); ++j) {
    for (std::size_t i = 1; i < w_ - 1; ++i) { 
      int gx = 0;
      int gy = 0;

      for (int ky = -1; ky <= 1; ++ky) {
        std::uint8_t pixel = input_data[((j + ky) * w_) + (i - 1)];
        gx += pixel * kSobelX.at(ky + 1).at(0);
        gy += pixel * kSobelY.at(ky + 1).at(0);

        pixel = input_data[((j + ky) * w_) + (i)];
        gx += pixel * kSobelX.at(ky + 1).at(1);
        gy += pixel * kSobelY.at(ky + 1).at(1);

        pixel = input_data[((j + ky) * w_) + (i + 1)];
        gx += pixel * kSobelX.at(ky + 1).at(2);
        gy += pixel * kSobelY.at(ky + 1).at(2);
      }

      auto magnitude = static_cast<int>(round(std::sqrt((gx * gx) + (gy * gy))));
      output[(j * w_) + i] = (magnitude > input.threshold) ? magnitude : 0;
    }
  }
}, tbb::simple_partitioner());
```
