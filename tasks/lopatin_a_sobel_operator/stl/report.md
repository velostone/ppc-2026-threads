## 1. Контекст

Решается задача выделения ребер на изображении с использованием оператора Собеля. Математическая постановка и последовательный
алгоритм описаны в файле отчета `seq/report.md`.
Алгоритм хорошо распараллеливается ввиду отсутствия зависимостей по данным между итерациями циклов. Предлагается организация
параллелизма с помощью потоков `std::thread`.
Данный подход позволяет вручную контролировать работу потоков в параллельной области, что полезно для сложных задач,
с которыми OMP и TBB не справляются или порождают дополнительные накладные расходы.

## 2. Постановка задачи

На вход алгоритма подается изображение типа `Image`, пиксели которого упакованы в `std::vector<uint8_t>`. На выходе
ожидается результирующий вектор `std::vector<int>`, компоненты которого принимают значение 0, если ребро отсутствует,
и `magnitude`, если ребро есть. За деталями обращаться к файлу `seq/report.md`.

## 3. Базовый алгоритм

Последовательная версия алгоритма в цикле по изображению обходит внутренние пиксели, применяя к ним ядра оператора Собеля
и вычисляя нормы соответствующих градиентов. Решение о наличии ребра принимается в результате сравнения значения нормы
с некоторым заданным порогом чувствительности `threshold`. За деталями обращаться к файлу `seq/report.md`.

## 4. Схема распараллеливания

Набор итераций внешнего цикла (строки изображения) вручную делится на порции `portion` размера `rows / n_threads`,
где `/` - деление нацело. Формируется вектор потоков `std::vector<std::thread>`, в который по мере создания добавляются
нити, запускаемые для выполнения функции `RunSobel()`. Функция `RunSobel()` является оберткой над телом внешнего цикла,
принимает входные и выходные данные, а также набор итераций `[start, end)`, и выполняет алгоритм Собеля для переданного
диапазона. Данные распределяются равномерно с точностью до размера хвоста. Все потоки пишут непосредственно в `output`,
при этом гонки данных не происходит, так как запись ведется по непересекающимся адресам.
Окончание выполнения функции `RunSobel()` явно синхронизируется методом `join()`, вызываемым для всех потоков.
Это обеспечивает минимальные накладные расходы и приводит к корректному результату.
Если число строк не делится на число потоков нацело, то остаток размера `rows % n_threads` итераций выполняет основной
поток после параллельной секции.

## 5. Детали реализации

- Файлы: stl/include/ops_stl.hpp, stl/src/ops_stl.cpp
- Каждый поток в параллельном режиме выполняет `rows / n_threads` итераций, основной поток дополнительно обрабатывает
хвост размера `rows % n_threads`
- Если число доступных в ОС потоков больше числа строк изображения, то создается столько потоков, сколько строк доступно
для обработки
- Размещение потоков в векторе осуществляется методом `emplace_back()`
- Детали реализации непосредственного алгоритма оператора Собеля в его последовательной версии описаны в файле `seq/report.md`

## 9. Выводы

При решении задачи выделения ребер на изображении с помощью оператора Собеля параллелизм, организованный вручную с помощью
потоков `std::thread`, дал сублинейное ускорение до 5.6 раз при 6 потоках, что соответствует числу логических ядер процессора.
Наибольшая эффективность 0.99 была достигнута на 2 потоках. Дальнейшее масштабирование привело к падению эффективности
до 0.38 при 12 потоках.
По сравнению с OMP и TBB реализациями наблюдалось небольшое замедление для 4 и более потоков, что может быть вызывано
неэффективной обработкой хвоста после явной синхронизации (главный поток будет простаивать в ожидании остальных
потоков группы).

## Приложение

```cpp
// File: stl/src/ops_stl.cpp
const auto &input = GetInput();
auto &output = GetOutput();

const auto rows = input.height - 2;
const auto n_threads = std::min(static_cast<std::size_t>(ppc::util::GetNumThreads()), rows);

std::vector<std::thread> threads;
threads.reserve(n_threads);

const std::size_t portion = rows / n_threads;
const std::size_t tail = rows % n_threads;

for (std::size_t i = 0; i < n_threads; ++i) {
  std::size_t start = 1 + (i * portion);
  std::size_t end = 1 + ((i + 1) * portion);
  threads.emplace_back(RunSobel, std::cref(input), start, end, std::ref(output));
}

for (auto &thread : threads) {
  if (thread.joinable()) {
    thread.join();
  }
}

if (tail != 0) {
  std::size_t start = input.height - 1 - tail;
  RunSobel(input, start, input.height - 1, output);
}
```
